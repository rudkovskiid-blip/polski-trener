# Польский тренажёр — подготовка к экзамену на ПМЖ

Офлайн-тренажёр (PWA) для подготовки к устному экзамену по польской культуре
(история, география, символы, традиции, знаменитые поляки + личный блок).
Уровень вопросов — как у Карты поляка. Ставится на iPhone и Mac, работает офлайн,
без сервера и аккаунта. Все данные — только на устройстве.

## Что внутри

- **109 карточек** билингва (PL/RU) + транскрипция + «почему» + источник:
  общие/символы 21, история 41, география 13, традиции 11, знаменитые поляки 10,
  современная Польша 6, личный блок 7.
- **Учить** — active recall (ответь вслух → проверь себя), интервальные повторения
  (FSRS, как в Anki), чередование тем, фильтр категорий, озвучка 🔊.
- **Экзамен** — N случайных вопросов под таймер → счёт и слабые темы.
- **Прогресс** — освоение по темам, точность, история экзаменов, бэкап JSON.
- **Личный блок** — шаблоны ответов («почему Польша», про деда и т.д.),
  редактируются и сохраняются на устройстве.

## Запуск локально

```bash
npm install
npm run dev      # http://localhost:5173
```

Сборка прод-версии:

```bash
npm run build    # результат в dist/
npm run preview  # локальный просмотр сборки
```

## Установка как приложение

- **iPhone (Safari):** открыть сайт → «Поделиться» → «На экран Домой».
- **Mac (Safari/Chrome):** «Файл/⋮» → «Добавить в Dock» / «Установить приложение».

## Деплой на GitHub Pages

1. Создать репозиторий на GitHub и запушить этот код в ветку `main`.
2. В репозитории: **Settings → Pages → Build and deployment → Source: GitHub Actions**.
3. Воркфлоу `.github/workflows/deploy.yml` соберёт и опубликует приложение
   (база пути выставляется автоматически из имени репозитория).
4. Адрес будет вида `https://<логин>.github.io/<репозиторий>/` — его и открыть на iPhone.

## Сквозной прогресс между устройствами (аккаунт + синхронизация)

По умолчанию прогресс хранится локально (IndexedDB). Чтобы он синхронизировался
между компом и iPad под одним аккаунтом, подключается бесплатный Supabase.
Без ключей приложение работает офлайн как раньше — панель «Аккаунт» просто скрыта.

**Настройка (один раз, ~5 минут):**

1. Зарегистрируйся на [supabase.com](https://supabase.com) → **New project**
   (регион ближе к тебе, например Frankfurt). Придумай пароль базы — он не нужен
   в приложении, просто сохрани.
2. Открой **SQL Editor → New query**, вставь содержимое [`supabase/schema.sql`](supabase/schema.sql)
   и нажми **Run** — создадутся таблицы и политики безопасности (RLS).
3. **Безопасность — чтобы чужой не завёл аккаунт:** сначала заведи свой аккаунт
   (см. п.5), а затем **Authentication → Sign In / Providers → Email** выключи
   **«Allow new users to sign up»** и сохрани. После этого регистрация закрыта —
   войти можно только существующим аккаунтом.
4. Возьми ключи: **Project Settings → API →** `Project URL` и `anon public` key.
   - Локально: скопируй `.env.example` в `.env.local` и подставь их.
   - На GitHub Pages: **Settings → Secrets and variables → Actions → New repository secret**,
     добавь `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`, затем перезапусти деплой.
5. **Заведи свой аккаунт** (пока регистрация ещё открыта): проще всего в Supabase
   **Authentication → Users → Add user** (email + пароль, отметить email как
   подтверждённый). Либо временно включи sign-up, войди в приложении и выключи обратно.
6. Открой приложение → вкладка **Прогресс** → войди по email и паролю. Дальше прогресс
   синхронизируется автоматически; кнопка «Синхронизировать сейчас» — принудительно.

Как работает синхронизация: каждая правка пишется локально и уходит в облако;
при входе/открытии выполняется двустороннее слияние (выигрывает более свежая
запись, экзамены объединяются). Офлайн-правки подхватятся при следующем открытии.

## Данные и регенерация банка

- Готовый банк: `src/data/cards.json` (фактические карточки) + `src/data/personal.ts` (личный блок).
- Исходники по категориям: `content-staging/*.json`.
- Пересобрать `cards.json` из staging: `node scripts/merge-cards.mjs`.
- Перегенерировать иконки: `npm run icons`.

## Стек

React 18 + TypeScript + Vite 6, vite-plugin-pwa (офлайн), IndexedDB (idb),
ts-fsrs (интервалы), zustand (состояние), Web Speech API (озвучка pl-PL).
